# 视频逐字稿整理技能

## 技能用途

将口语化的视频逐字稿整理成结构清晰、易于阅读的书面文档，使用户无需观看视频即可完整吸收内容。

## 使用场景

当用户提供以下内容时触发此技能：
- 微头条写作培训逐字稿
- 视频转文字稿需要整理
- 直播/录课的即兴讲解内容需要书面化
- 口语化教学内容的清理和优化
- 用户明确要求"整理逐字稿"、"清理口语内容"、"变成书面语"等

## 交互流程

1. **用户提供日期**：如"12.15"、"12.16"等，作为初始文件名
2. **用户提供逐字稿内容**
3. **AI整理内容**
4. **自动保存为Markdown文件**：保存到当前工作目录，文件名为 `{日期}.md`
5. **告知用户**：文件已保存，简述新增内容板块
6. **用户告知完成**：用户说"完成了"、"发完了"等表示当日内容结束
7. **AI总结并重命名**：
   - 提取当日内容核心主题
   - 总结为10字以内
   - 重命名文件为 `{日期}{10字总结}.md`，如 `12.15结营直播.md`
8. **告知用户**：文件已重命名

## 智能识别规则

收到逐字稿后，首先判断内容类型：

| 内容类型 | 识别特征 | 处理方式 |
|---------|---------|---------|
| **方法论/套路** | 出现"步骤是"、"方法是"、"模板"、"公式"等词 | 理论说明 + 可填空模板 |
| **案例拆解** | 出现"这篇文章"、"你看这个"、"举个例子" | 拆解过程 + 结论 + 分析框架 |
| **避坑指南** | 出现"不要"、"千万别"、"避免"、"误区" | 醒目标注（⚠️）+ 就近放置 |
| **学员问答** | 有人提问、老师回应"这个问题问得好" | 完整保留 Q&A 格式 |
| **核心观点** | 老师重复强调、放慢语速、语气加重 | **加粗** + 单独提炼 |

## 基础清理

删除以下冗余内容：
- **语气词**：嗯、啊、呃、那个、就是、对吧、兄弟们等
- **重复表述**：如"你要那什么呀？你要那什么呀？"只保留一次
- **卡顿和自我纠正**：如"你你你"、"我我我"
- **无意义的口头禅和拖延词**

## 结构优化

将碎片化口语整理为完整书面语：
- 补充完整句子结构
- 按知识点/案例合理分段
- 为每段添加概括性小标题
- 大标题标注序号（如 1.、2.、3.）

## 输出格式模板

```markdown
---
[本次整理] {简要说明本次整理了什么内容}  #{关键词1} #{关键词2}
---

## {序号}. {段落小标题}

### 核心观点
**重点内容加粗标注**

### 方法论/套路（如有）
**原理**：{为什么这样做}

**模板**：
```
可填空格式
```

> 示例：
> {老师举的具体例子}

⚠️ 避坑：{需要注意的问题}

### 案例拆解（如有）
**原文**：{文章/案例简介}

**拆解过程**：
1. {第一步分析}
2. {第二步分析}

**为什么火**：{核心原因}

**可复用方法**：{以后怎么用}

### 学员问答（如有）
**Q**：{学员问题}
**A**：{老师解答}
```

## 特殊场景处理

| 场景 | 处理方式 |
|-----|---------|
| 内容不完整（分段发送） | 立即整理当前部分，保持格式一致性，便于后续拼接 |
| 老师讲重复内容 | 第一次详细整理，后续标注"（与第X部分重复，略）" |
| 口语混乱严重 | 按逻辑重新组织，不追求逐字对应 |
| 有代码/具体文本 | 用代码块或引用块独立呈现 |

## 分段处理流程

```
收到逐字稿
    ↓
1. 询问日期标识（如"12.15"）
2. 自动识别内容类型
3. 提取关键词标签
4. 按类型匹配处理方式
5. 输出整理结果
    ↓
头部说明：[本次整理] 内容概要  #标签1 #标签2
    ↓
6. 自动保存为Markdown文件（{日期}.md）
```

## 自动保存规则

**必须自动保存整理后的内容**：

1. **初始文件命名**：使用用户提供的日期作为文件名，如 `12.15.md`

2. **保存位置**：当前工作目录

3. **保存时机**：
   - 首次整理：创建新文件并保存
   - 追加内容：读取现有文件，追加新内容到末尾

4. **保存后反馈**：告知用户文件已保存，简要说明新增内容板块

5. **完成后重命名**：
   - 当用户说"完成了"、"发完了"、"就这些"等表示结束时
   - 提取当日内容核心主题
   - 总结为**10字以内**
   - 重命名文件为 `{日期}{总结}.md`，如 `12.15结营直播.md`、`12.16热点写作.md`
   - 告知用户新文件名

每次收到逐字稿后：
1. 询问或确认日期标识（如果是首次）
2. 立即开始整理当前部分
3. 在输出头部说明本次整理了什么
4. 添加相关关键词标签
5. 保持格式一致性，便于后续拼接
6. **自动保存到文件**（新增）
