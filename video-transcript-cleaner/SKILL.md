# 视频逐字稿整理技能

## 技能用途

将口语化的视频逐字稿整理成结构清晰、易于阅读的书面文档，使用户无需观看视频即可完整吸收内容。

## 使用场景

当用户提供以下内容时触发此技能：
- 微头条写作培训逐字稿
- 视频转文字稿需要整理
- 直播/录课的即兴讲解内容需要书面化
- 口语化教学内容的清理和优化
- 用户明确要求"整理逐字稿"、"清理口语内容"、"变成书面语"等

## 交互流程

1. **用户提供日期**：如"12.15"、"12.16"等，作为初始文件名
2. **用户提供逐字稿内容**
3. **AI整理内容**
4. **自动保存为Markdown文件**：保存到当前工作目录，文件名为 `{日期}.md`
5. **告知用户**：文件已保存，简述新增内容板块
6. **用户告知完成**：用户说"完成了"、"发完了"等表示当日内容结束
7. **AI总结并重命名**：
   - 提取当日内容核心主题
   - 总结为10字以内
   - 重命名文件为 `{日期}{10字总结}.md`，如 `12.15结营直播.md`
8. **告知用户**：文件已重命名

## 智能识别规则

收到逐字稿后，首先判断内容类型：

| 内容类型 | 识别特征 | 处理方式 |
|---------|---------|---------|
| **方法论/套路** | 出现"步骤是"、"方法是"、"模板"、"公式"等词 | 理论说明 + 可填空模板 |
| **案例拆解** | 出现"这篇文章"、"你看这个"、"举个例子" | 拆解过程 + 结论 + 分析框架 |
| **避坑指南** | 出现"不要"、"千万别"、"避免"、"误区" | 醒目标注（⚠️）+ 就近放置 |
| **学员问答** | 有人提问、老师回应"这个问题问得好" | 完整保留 Q&A 格式 |
| **核心观点** | 老师重复强调、放慢语速、语气加重 | **加粗** + 单独提炼 |

## 基础清理

删除以下冗余内容：
- **语气词**：嗯、啊、呃、那个、就是、对吧、兄弟们等
- **重复表述**：如"你要那什么呀？你要那什么呀？"只保留一次
- **卡顿和自我纠正**：如"你你你"、"我我我"
- **无意义的口头禅和拖延词**

## 结构优化

将碎片化口语整理为完整书面语：
- 补充完整句子结构
- 按知识点/案例合理分段
- 为每段添加概括性小标题
- 大标题标注序号（如 1.、2.、3.）

```markdown
---

---

## {序号}. {段落小标题}

### 核心观点
**重点内容加粗标注**

### 方法论/套路（如有）
**原理**：{为什么这样做}

**模板**：
```
可填空格式
```

> 示例：
> {老师举的具体例子}

⚠️ 避坑：{需要注意的问题}

### 案例拆解（如有）
**原文**：{文章/案例简介}

**拆解过程**：
1. {第一步分析}
```

## 特殊场景处理

| 场景 | 处理方式 |
|-----|---------|
| 内容不完整（分段发送） | 立即整理当前部分，保持格式一致性，便于后续拼接 |
| 老师讲重复内容 | 第一次详细整理，后续标注"（与第X部分重复，略）" |
| 口语混乱严重 | 按逻辑重新组织，不追求逐字对应 |
| 有代码/具体文本 | 用代码块或引用块独立呈现 |
| 长逐字稿（>15,000字） | 自动启动智能分段模式，按知识点切分后逐段整理 |

## 智能分段配置

**当逐字稿超过 15,000 字时，自动启动智能分段模式**

### 分段参数

- **启动阈值**：15,000 字（超过此字数自动分段）
- **目标段落长度**：10,000 字
- **段落长度范围**：8,000-12,000 字
- **最小段落长度**：5,000 字
- **扫描窗口大小**：4,000 字

### 分段算法

采用"滑动窗口 + 边界识别"方式：

1. **字数检测**：统计逐字稿总字数
2. **分段判断**：如果超过 15,000 字，自动启动智能分段
3. **智能切分**：
   - 当累计字数达到 8,000 字后，在接下来 4,000 字范围内寻找最佳分段点
   - 按边界优先级选择分段位置（见下方）
4. **逐段整理**：依次整理每个段落，实时输出整理结果
5. **合并保存**：将所有段落合并为单个 markdown 文件

### 分段边界优先级

从高到低：

1. **内容类型切换点**：方法论→案例→问答→避坑
2. **过渡话语**：
   - 开启新话题："好，接下来我们讲"、"那么下一个"、"我们再看"、"现在来说说"、"第X个/另一个"
   - 总结收尾："所以说"、"总结一下"、"简单来说就是"、"这就是"、"大概就是这样"
   - 案例切换："我们看个例子"、"举个案例"、"你看这篇文章"、"再来一个"
3. **段落自然结束点**：总结性语句后的停顿

### 内容类型切换检测

通过关键词密度变化判断：

- **方法论区域**：高频出现"步骤"、"方法"、"模板"、"公式"
- **案例区域**：高频出现"这篇"、"你看"、"举例"、"比如"
- **问答区域**：出现"有同学问"、"这个问题"、"Q:"、"A:"
- **避坑区域**：高频出现"不要"、"别"、"避免"、"误区"

### 边界情况处理

| 情况 | 处理方式 |
|-----|---------|
| 逐字稿在临界值附近 | 14,500字不分段；15,500字分段 |
| 最后一段过短 | 如果 < 5,000字，合并到倒数第二段 |
| 无法找到合适分段点 | 在12,000字位置强制分段，下一段开头标注"（接上段）" |
| 用户中途追加内容 | 每次独立判断是否需要分段 |

### 逐段展示格式

```markdown
检测到逐字稿约 42,000 字，已自动分为 4 段处理

---
[第 1/4 段整理] 开场+选题方法论  #选题 #方法论
---

## 1. 选题的底层逻辑
[完整整理内容]

---
[第 2/4 段整理] 标题公式+案例拆解  #标题 #案例
---

## 2. 标题的三要素
[完整整理内容]

...（依次输出第3、4段）

---
✅ 全部整理完成，已保存至：12.15.md
```

### 段落衔接规则

- **序号连续**：第1段最后是"## 3."，第2段从"## 4."开始
- **避免重复**：分段点在知识点中间时，第2段开头标注"（接上段）"
- **统一头部**：最终合并文件时，生成统一的总头部说明：`[本次整理] 完整逐字稿整理（共X段，约X万字） #标签1 #标签2`

## 分段处理流程

```
收到逐字稿
    ↓
1. 询问日期标识（如"12.15"）
2. 字数检测（是否 > 15,000字）
    ↓
   【是】→ 智能分段模式
   - 预处理：基础清理 + 标记分段点
   - 生成分段方案
   - 逐段整理并实时输出
   - 合并保存为单文件
    ↓
   【否】→ 常规模式
   - 自动识别内容类型
   - 提取关键词标签
   - 按类型匹配处理方式
   - 输出整理结果
    ↓
头部说明：[本次整理] 内容概要  #标签1 #标签2
    ↓
3. 自动保存为Markdown文件（{日期}.md）
```

## 自动保存规则

**必须自动保存整理后的内容**：

1. **初始文件命名**：使用用户提供的日期作为文件名，如 `12.15.md`

2. **保存位置**：当前工作目录

3. **保存时机**：
   - 首次整理：创建新文件并保存
   - 追加内容：读取现有文件，追加新内容到末尾

4. **保存后反馈**：告知用户文件已保存，简要说明新增内容板块

5. **完成后重命名**：
   - 当用户说"完成了"、"发完了"、"就这些"等表示结束时
   - 提取当日内容核心主题
   - 总结为**10字以内**
   - 重命名文件为 `{日期}{总结}.md`，如 `12.15结营直播.md`、`12.16热点写作.md`
   - 告知用户新文件名

每次收到逐字稿后：
1. 询问或确认日期标识（如果是首次）
2. 立即开始整理当前部分
3. 在输出头部说明本次整理了什么
4. 添加相关关键词标签
5. 保持格式一致性，便于后续拼接
6. **自动保存到文件**（新增）
